# CC Desktop Switch v1.1.0

## 中文

v1.1.0 是一次桌面运行时升级版本：应用主体迁移到 Tauri + Rust，同时保留原有前端界面和提供商配置逻辑。目标是提升启动速度、窗口恢复、托盘行为和本机代理稳定性，同时避免重新设计用户已经熟悉的界面。

### 主要变化

- **迁移到 Tauri + Rust 桌面运行时**：
  - 新增 Tauri v2 应用壳和 Rust 命令层，替代原先较重的桌面承载方式。
  - 保留原有 HTML/CSS/JavaScript 界面，不改变 Provider 管理、设置页和一键应用的主流程。
  - Windows 发布链路已切换到 Tauri/Rust 产物，旧 Python 兼容运行时已退出当前主路径。
- **提升桌面体验**：
  - 优化启动加载链路，减少启动时长时间空白。
  - 修复关闭窗口后隐藏到托盘，再次点击托盘无法唤回窗口的问题。
  - Provider 卡片启用成功后会询问是否重启 Claude Desktop；如果 Claude 未运行，会直接打开。
- **完善 Provider 配置行为**：
  - 恢复 Tauri 路径中的 Xiaomi MiMo、Xiaomi MiMo Token Plan、阿里云百炼 Token Plan 和第三方模型入口。
  - 连接测试改为只判断服务器地址可达；API Key 是否可用由获取模型或模型检测给出明确提示。
  - 获取模型列表时只自动填充 Default 映射，其余模型槽位由用户手动填写或回退到 Default。
  - 一键应用流程改为应用内确认弹窗，避免原生确认框在 Tauri WebView 中无反馈。
- **新增反馈入口**：
  - Dashboard 和设置页新增反馈入口。
  - 支持提交文字、附件和脱敏诊断信息，便于定位问题。
- **补充 Rust 工具链清理**：
  - 前端 app/style bundle、静态契约检查、共享模型槽位契约、Windows release 资产整理和 macOS 包装编排已收敛到 Rust `xtask`。
  - 共享模型槽位、Provider 映射兼容规则和 Tauri bridge 契约由 Rust 源生成，减少前端和运行时之间的重复维护。
  - JavaScript 保留为 Tauri WebView 的界面层，Provider 诊断、模型获取、配置读写、桌面配置、反馈提交、更新和代理转发等业务逻辑由 Rust 负责。
- **本地验证补充**：
  - 已通过静态契约检查、前端语法检查、Rust 格式化、`xtask` 编译和测试、Tauri Rust 编译和测试、前端构建，以及 macOS 未签名 `.app` 构建。
  - 用户已手动验证本地 App 可用。
  - 本次补充不提升版本号，也不替换既有安装包产物。

## English

v1.1.0 upgrades the desktop runtime to Tauri + Rust while keeping the existing provider-management UI and configuration flow. The release focuses on startup behavior, window restoration, tray behavior, local proxy stability, and provider-management correctness without redesigning the app.

### Highlights

- **Tauri + Rust desktop runtime**:
  - Added a Tauri v2 shell and Rust command layer for the desktop app.
  - Preserved the existing HTML/CSS/JavaScript interface and the established Provider, Settings, and Apply workflows.
  - Moved the Windows release chain to Tauri/Rust artifacts and retired the old Python compatibility runtime from the current path.
- **Desktop experience improvements**:
  - Optimized the startup loading chain to reduce blank-screen delay.
  - Fixed tray restoration after closing the window into the background.
  - Provider enable now asks whether to restart Claude Desktop; if Claude is not running, the app opens it directly.
- **Provider workflow fixes**:
  - Restored Xiaomi MiMo, Xiaomi MiMo Token Plan, Alibaba Cloud Bailian Token Plan, and the generic third-party preset in the Tauri path.
  - Connection test now checks URL reachability only; model discovery and model checks show explicit API key failures.
  - Model discovery only fills the Default mapping and leaves other slots manual or fallback-to-Default.
  - One-click apply now uses an in-app confirmation modal instead of native browser confirmation.
- **Feedback flow**:
  - Added feedback entry points in Dashboard and Settings.
  - Supports text, attachments, and sanitized diagnostics for issue triage.
- **Rust tooling cleanup**:
  - Frontend app/style bundling, static contract checks, shared model-slot contracts, Windows release asset assembly, and macOS packaging orchestration now run through Rust `xtask`.
  - Shared model slots, Provider mapping compatibility rules, and Tauri bridge contracts are generated from Rust-owned sources to reduce frontend/runtime duplication.
  - JavaScript remains the Tauri WebView UI layer, while Provider diagnostics, model discovery, config storage, Desktop configuration, feedback submission, updates, and proxy forwarding are handled by Rust.
- **Local validation update**:
  - Static contract checks, frontend syntax checks, Rust formatting, `xtask` build/tests, Tauri Rust build/tests, frontend build, and unsigned macOS `.app` build have passed locally.
  - The local app has been manually verified by the user.
  - This note does not bump the version and does not replace existing installer artifacts.
