# CC Desktop Switch v1.1.1

## 中文

v1.1.1 是一个 macOS 配置修复版本，重点修复 Tauri/Rust 迁移后 Claude Desktop 1M 上下文模型能力声明没有正确落到当前生效配置的问题。

### 主要变化

- **修复 macOS 1M 上下文配置写入**：
  - macOS 写入 `claude_desktop_config.json` 和当前生效的 `configLibrary` 条目时，现在会保留 `inferenceModels` 里的 `supports1m: true`。
  - 修复一键应用后仍提示“1M 上下文模型尚未写入桌面版配置”的问题。
  - DeepSeek 1M 和通义千问 1M 这类依赖 `supports1m` 的配置，在重新一键应用并重启 Claude Desktop 后会被健康检查正确识别。
- **补充回归覆盖**：
  - 更新 macOS JSON/configLibrary 写入测试，确认模型对象元数据不会再被压平成普通字符串数组。
  - 保留现有 Tauri/Rust 运行时和前端界面，不改变 Provider 管理、模型映射和一键应用工作流。

### 升级后建议

如果你之前遇到 1M 上下文提示一直存在，请升级后重新点击“一键应用到 Claude 桌面版”，然后完全重启 Claude Desktop。

### 下载建议

- macOS 用户可以下载 `CC-Desktop-Switch-v1.1.1-macOS-arm64.pkg` 或 `CC-Desktop-Switch-v1.1.1-macOS-arm64.dmg`。
- Windows 用户可以继续使用 Release 页面中的 Windows 安装版或便携版。

## English

v1.1.1 is a macOS configuration hotfix. It fixes the 1M context capability metadata written to Claude Desktop after the Tauri/Rust migration.

### Highlights

- **Fixed macOS 1M context configuration writes**:
  - macOS writes now preserve `supports1m: true` inside `inferenceModels` for both `claude_desktop_config.json` and the active `configLibrary` entry.
  - Fixes the warning that 1M context models have not been written even after applying a Provider.
  - DeepSeek 1M and Qwen 1M configurations are recognized correctly after reapplying and fully restarting Claude Desktop.
- **Regression coverage**:
  - Updated macOS JSON/configLibrary tests to ensure model metadata is not flattened into plain model-name arrays.
  - Keeps the existing Tauri/Rust runtime and provider-management UI unchanged.

### After Updating

If the 1M context warning was already visible, apply the Provider to Claude Desktop again and fully restart Claude Desktop after installing this version.

### Downloads

- macOS users can choose `CC-Desktop-Switch-v1.1.1-macOS-arm64.pkg` or `CC-Desktop-Switch-v1.1.1-macOS-arm64.dmg`.
- Windows users can continue using the Windows installer or portable build from the Release page.
